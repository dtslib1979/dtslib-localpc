# 모바일 허브 제어 아키텍처

> 유선 없음. WiFi = 폰 핫스팟/테더링 전제.

---

## 전체 구조

```
PC (WSL)
  │
  └─ Tailscale (인터넷)
       │
       └─ 폰 S25 Ultra  ← 허브
            │
            ├─ [WiFi 핫스팟]─────────────────────────────┐
            │    │                                       │
            │    ├─ Android 기기 (차량, 태블릿)           │
            │    │    └─ ADB ON → PC에서 완전 제어 ✅     │
            │    │                                       │
            │    ├─ ESP32 / IoT 기기                     │
            │    │    └─ WiFi 연결 → HTTP/MQTT → 제어 ✅  │
            │    │                                       │
            │    └─ 일반 기기 (노트북 등)                  │
            │         └─ SSH → 제어 ✅                   │
            │                                            │
            └─ [Bluetooth]                               │
                 │                                       │
                 ├─ 페어링된 기기                         │
                 │    └─ SSH → 폰 BT 명령 → 간접 제어 △  │
                 │                                       │
                 └─ BLE 센서                              │
                      └─ 폰이 브릿지 → 간접 제어 △        │
```

---

## 스텝다운 판단 기준

```
제어 대상
  │
  ├─ 폰 핫스팟 WiFi에 붙는가?
  │    ├─ YES → IP 잡힘 → PC에서 직접 제어 ✅
  │    └─ NO
  │         └─ 블루투스로 폰에 페어링 가능한가?
  │               ├─ YES → 폰 경유 간접 제어 △
  │               └─ NO → 제어 불가 ❌
```

---

## 기기별 조건표

| 기기 | 조건 | 방법 | 수준 |
|------|------|------|------|
| Android 기기 (차량, 태블릿) | WiFi + ADB ON | adb shell | ✅ 완전 제어 |
| ESP32 / IoT | WiFi 연결 | HTTP/MQTT | ✅ 완전 제어 |
| 일반 PC/서버 | WiFi 연결 | SSH | ✅ 완전 제어 |
| 블루투스 기기 | 폰에 페어링 | SSH → 폰 BT 명령 | △ 간접 제어 |
| BLE 센서 | 폰이 브릿지 | 폰 BLE 스캔 | △ 간접 제어 |
| WiFi 미지원 기기 | — | — | ❌ 불가 |

---

## 핵심 원칙

> **WiFi 핫스팟에 붙는 순간 PC 통제권 안으로 들어옴.**
> ESP32든 차량이든 태블릿이든 조건은 동일.

폰 = 모바일 공유기 겸 ADB 허브.
PC는 Tailscale로 항상 폰에 붙어있으므로 — 박씨가 어디 있든 폰 핫스팟 네트워크 전체가 PC 통제권.
