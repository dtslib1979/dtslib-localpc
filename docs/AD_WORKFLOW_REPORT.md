## Q1. 어떻게 리팩토링할 거냐

### 한 줄
**"티켓 단위로, 삼각형 OPS 가, Papyrus BEFORE/AFTER 남기면서, 박씨 게이트 2회만 거치고 자율 진행."**

### 단계 (티켓 1개당)

```
1. 박씨 발화 1줄
   "T-NNNN 처리해" 또는 "펜딩에서 알아서 골라"
        ↓
2. Claude (Architect)
   - 옆 phone_aider/tab_aider 세션 tmux capture (이미 가능)
   - 메모리 99건 + 7패키지 맵 + 헌법 자동 로드
   - 관련 파일 grep + git log 자동
   - TICKET CONTEXT 자동 작성 (박씨 복붙 0)
        ↓
3. DeepSeek (Designer) 위임
   - Claude 가 프롬프트 자동 작성 → tmux send-keys
   - 체인 설계 + 커뮤니티 패턴 2~3 소스 인용
        ↓
4. Claude — Plan 통합
   - DeepSeek 응답을 박씨용 한국어 Plan 으로 정리
   - 헌법 § 5 절대 금지 자가 검사
        ↓
5. ★ 박씨 게이트 1 — Plan 결재 (1분)
        ↓ OK
6. Aider (Executor) 위임
   - Claude 가 Aider 인스트럭션 자동 작성 → tmux send-keys
   - Aider 가 diff-edit 정밀 수정 + 테스트 실행
        ↓
7. Claude — 자가 검수
   - tmux capture-pane 으로 결과 자동 캡처
   - git diff 기반 6 항목 자가 검증 (헌법/scope/mock/lint/타입/테스트)
   - 실패 시 자동 재시도 (최대 3회)
        ↓
8. Papyrus 자동 기록
   - dtslib-papyrus/ad/<티켓>/NOTES.md
   - BEFORE_REF / AFTER_REF (git diff)
   - METRICS.json (LOC/시간/비용/박씨 개입)
        ↓
9. ★ 박씨 게이트 2 — Commit 결재 (1분)
        ↓ OK
10. push + 00_INDEX.md 갱신 + 다음 티켓
```

**박씨 부담:** 티켓당 2분 (게이트 2회 × 1분).
**OPS 부담:** 30분~몇 시간 자율.

## Q2. 삼지사방 흩어진 코드/아이디어 어떻게 종합

### 현재 산재 상황 (5개월 결과)

| 위치 | 내용 |
|---|---|
| 28 레포 5,800+ 커밋 | 코드 본체 |
| 메모리 99건 | 박씨 결정/패턴/금지/펜딩 |
| 헌법 4종 (글로벌+레포별) | 운영 규칙 |
| 백서/리포트 다수 | 5개월 회고/정수 평가/워크센터/CLAUDE_OPS |
| 펜딩 16건 | 1줄 블로커로 멈춘 작업 |
| 박씨 머릿속 | 미문서화 아이디어 |

### 종합 메커니즘 4단계

#### 1단계 — 마스터 인덱스 (자동 생성)
**위치:** `dtslib-papyrus/ad/00_INDEX.md`

OPS 가 자동 작성:
- 28 레포 × 7패키지 × 99 메모리 → 1개 마스터 표
- 펜딩 16건 → 티켓 16개 자동 변환
- 우선순위 자동 매기기 (1줄 블로커 P0 / 펜딩 P1 / 신규 P2)

#### 2단계 — 키워드 라우팅 (이미 박힘)
박씨 메모리 `~/CLAUDE.md` 7패키지 키워드 자동 매핑:
- "텔레그램" → P0 TELEGRAM
- "유튜브" → P2 YOUTUBE
- "방송국" → P3 BROADCAST
- ...

박씨 한 단어 → OPS 가 해당 패키지 레포 전부 열고 작업 환경 세팅.

#### 3단계 — 메모리 자가 갱신
티켓 끝날 때마다 OPS 가:
- 새 패턴 발견 → 메모리 추가 (`feedback_*`)
- 펜딩 해소 → 메모리 마커 제거
- 충돌 발견 → 메모리 갱신

박씨가 직접 메모리 안 만져도 됨.

#### 4단계 — 박씨 머릿속 → STT → 메모리
박씨 STT 발화 = OPS 가 자동으로:
- 새 결정 → `project_*` 메모리 추가
- 새 룰 → `feedback_*` 메모리 추가
- 새 펜딩 → 마커 + 티켓화 자동 검토

### 결과
**5개월 산재 → 1개 마스터 인덱스 + 7패키지 라우팅 + 자동 갱신 메모리.** 박씨가 "어디 있더라" 안 찾아도 됨.

## Q3. 박씨 통제 위치 — 어디서 통제하나

### 박씨가 통제하는 5개 지점

| # | 지점 | 빈도 | 시간 |
|:--:|---|:--:|:--:|
| 1 | **티켓 입력** | 시작 시 | 30초 (1줄 발화) |
| 2 | **Plan 게이트** | 티켓당 1회 | 1분 (OK/NO) |
| 3 | **Commit 게이트** | 티켓당 1회 | 1분 (OK/NO) |
| 4 | **DAILY 확인** | 매일 1회 | 1분 (`ad/00_DAILY.md` 1줄 훑기) |
| 5 | **WEEKLY 회고 검수** | 주 1회 | 5분 (`ad/00_WEEKLY_RETRO.md`) |

### 박씨가 통제하지 않는 것 (OPS 자율)
- Explore (파일/메모리 검색)
- DeepSeek 위임 + 응답 처리
- Aider 인스트럭션 작성 + 실행
- 자가 검수 6 항목
- 자체 재시도 (3회 한도)
- 메트릭 수집
- INDEX/DAILY/WEEKLY 자동 작성

### 박씨 통제권 = "디렉터" 권한
- **방향**: 어떤 패키지 우선
- **거부권**: Plan/Commit NO 가능
- **롤백권**: 사후 git revert 명령
- **헌법 개정권**: CLAUDE_OPS § 수정

### 비유
박씨는 **영화 감독**. OPS 는 **촬영팀+편집팀+조명팀**.
감독은 "이 장면 찍어 / OK / NG" 만 결정. 카메라 직접 안 듦.

### 시간축

| 시점 | 박씨 | OPS |
|---|---|---|
| **0:00** | 티켓 입력 1줄 ("T-0001") | — |
| **0:01~0:05** | (자유 시간) | Explore + DeepSeek 위임 + Plan 작성 |
| **0:05** | Plan 1분 검토 → OK | (대기) |
| **0:06~0:30** | (자유 시간) | Aider 위임 + 검수 + 재시도 |
| **0:30** | Commit 1분 검토 → OK | (대기) |
| **0:31** | (다음 티켓 또는 자유) | push + 다음 티켓 |
| **매일 21:00** | DAILY 1분 확인 | (오늘 처리분 자동 작성) |
| **일요일** | WEEKLY 5분 검수 | (주간 회고 자동 작성) |

### 박씨 1주 부담 시뮬레이션 (7 티켓 처리 시)

| 활동 | 시간 |
|---|:--:|
| 티켓 입력 7회 × 30초 | 3.5분 |
| Plan 게이트 7회 × 1분 | 7분 |
| Commit 게이트 7회 × 1분 | 7분 |
| DAILY 7회 × 1분 | 7분 |
| WEEKLY 1회 × 5분 | 5분 |
| **총 박씨 부담 / 주** | **~30분** |

→ **주 30분 투입 = 7 티켓 처리.** 5주면 펜딩 16건 + 신규 19건 가능.
