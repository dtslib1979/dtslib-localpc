## 결정 요약

**Dell XPS 13 9370 (16GB) + D: 외장 HDD 2TB 구성 그대로 사용한다.**
parksy-audio(음악)와 parksy-image(웹툰) 두 프로젝트를 이 PC 하나에서 동시 운영한다.

---

## 왜 이게 최선인가

| 대안 | 비용 | 판정 |
|------|------|------|
| 클라우드 서비스 (AWS/GCP) | 월 변동비, GPU 쓰면 폭탄 | ❌ 지금은 안 됨 |
| 서버 PC 24시간 가동 | 전기세 + 장비값 | ❌ 비싸다 |
| 중고 워크스테이션 | ~150만원 | ❌ 추가 지출 아깝다 |
| **현재 Dell + 외장 HDD** | **0원** | **✅ 이걸로 간다** |

---

## 현재 구성

| 항목 | 스펙 | 역할 |
|------|------|------|
| PC | Dell XPS 13 9370 (i7-8550U, 16GB, NVMe 512GB) | 작업 본체 |
| D: | WD My Passport 2TB (USB 3.0 HDD) | 모든 작업 데이터 |
| 태블릿 | RustDesk 원격 접속 | 밖에서 PC GUI 확인 |
| 폰 | Termux Claude Code | 경량 작업 + 큐 등록 |

---

## 운영 방식

### 두 프로젝트 동시 운영

```
[PC Claude Code]     parksy-audio 렌더링 (FluidSynth + FFmpeg)
                     parksy-image Python 작업 (OpenCV, PSE 빌드)

[폰 Termux]          parksy-image Node.js 작업 (cli.js, 큐 등록)
                     경량 Git 작업
```

### RAM 사용 실측

```
Windows + 상시:       ~4.0 GB
parksy-audio 피크:    ~1.0 GB
parksy-image 피크:    ~0.4 GB
RustDesk:             ~0.1 GB
────────────────────────────
합계:                 ~5.5 GB / 16 GB
여유:                 ~10.5 GB  → 충분
```

### D: 외장 HDD 속도 문제

- USB 3.0 HDD는 NVMe 대비 30~50배 느리다
- FluidSynth SF2 로딩, WAV 쓰기가 느림
- **하지만 무인 배치라 상관없다** — 돌려놓고 나가면 됨
- 22곡 배치 ~2시간. 사람이 기다릴 일이 아님

### 이동성 유지

- D: 외장 HDD를 뽑아서 다른 PC에 꽂으면 동일 환경
- C: 드라이브는 PC마다 다르니까 작업 데이터를 C:에 두지 않는다
- 도구(Python, FFmpeg 등)만 각 PC에 설치하면 됨

### 최종 결과물 저장 전략

| 대상 | 저장소 | 비용 |
|------|--------|------|
| 코드 | GitHub | 무료 |
| 음악/영상 결과물 | YouTube 업로드 | 무료 |
| 이미지 결과물 | Google Drive | 무료 |
| D:\tmp 임시 파일 | 업로드 후 삭제 | — |

D: 용량은 1.4TB 여유. 월 ~8.5GB 증가 추정. 13년 이상 사용 가능.

## 돈이 생기면 — 업그레이드 경로

| 우선순위 | 업그레이드 | 예상 비용 | 효과 |
|----------|-----------|----------|------|
| 1순위 | D: 외장 SSD로 교체 (1~2TB) | 10~15만원 | 렌더링 I/O 속도 30배 향상 |
| 2순위 | NAS (Synology 등) | 30~50만원 | 중앙 저장소, 어디서든 접속 |
| 3순위 | 중고 워크스테이션 | 100~150만원 | CPU/GPU 파워, 다중 작업 |
| 4순위 | 클라우드 GPU (GCP/AWS) | 월 변동비 | AI 추론, 대량 렌더링 |

**지금은 1~4순위 전부 불필요. 현재 구성으로 충분히 돌아간다.**
